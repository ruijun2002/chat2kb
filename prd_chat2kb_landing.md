# Chat2KB 产品 Landing Page PRD

## 1. 产品定位

**Chat2KB** — 对话式数据库智能平台，通过自然语言与 KubeBlocks 全系产品互动。两大核心引擎：知识库问答 + Demo 环境运营洞察，让数据开口说话。

**一句话口号**：*Ask Your Database Anything*

---

## 2. 目标用户

| 角色 | 核心诉求 | 使用场景 |
|------|---------|---------|
| **售前工程师** | 快速回答客户技术问题 | KB 问答助手 |
| **销售代表** | 洞察客户试用行为，识别高意向线索 | Demo 环境运营助手 |
| **客户成功** | 监控客户活跃度，主动干预沉默客户 | Demo 环境运营助手 |
| **平台管理员** | 评估资源利用率，优化 Demo 环境成本 | Demo 环境运营助手 |
| **产品经理** | 分析产品使用数据，发现体验卡点 | Demo 环境运营助手 |
| **开发者/DBA** | 查询 KB 使用文档、最佳实践、故障排查 | KB 问答助手 |

## 5. 视觉设计系统

### 5.1 设计关键词

**科技感 × AI Native × 数据库硬核**

- **氛围**：深邃、未来、数据流动感
- **情绪**：智能、可信、高效
- **隐喻**：数据粒子、神经网络、光线流动

### 5.2 色彩系统

| 角色 | 色值 | 用途 |
|------|------|------|
| **主背景** | `#0A0E1A` 深空黑 | 页面主背景，营造科技深邃感 |
| **次背景** | `#111827` 石墨灰 | 卡片、区块背景 |
| **强调色** | `#00D4FF` 数据青 | 主品牌色，按钮、高亮、交互态 |
| **辅助色** | `#7C3AED` 神经紫 | 渐变、标签、次强调 |
| **成功色** | `#10B981` 翠影绿 | 状态指示、正向数据 |
| **警告色** | `#F59E0B` 琥珀橙 | 提醒、中度风险 |
| **文字主色** | `#F9FAFB` 星云白 | 主标题、正文 |
| **文字次色** | `#9CA3AF` 月灰 | 辅助说明、元信息 |

### 5.3 字体系统

| 层级 | 字体 | 大小 | 字重 | 用途 |
|------|------|------|------|------|
| **Display** | Inter / SF Pro Display | 72px | 800 | Hero 主标题 |
| **H1** | Inter | 48px | 700 | 区块标题 |
| **H2** | Inter | 32px | 600 | 功能标题 |
| **H3** | Inter | 24px | 600 | 卡片标题 |
| **Body** | Inter | 16px | 400 | 正文 |
| **Mono** | JetBrains Mono | 14px | 400 | 代码、数据展示 |
| **Caption** | Inter | 12px | 500 | 标签、元数据 |

### 5.4 动效语言

| 动效 | 参数 | 场景 |
|------|------|------|
| **数据流** | 线性流动，速度 20px/s，透明度 0.3→0.6 | 背景装饰线 |
| **脉冲呼吸** | scale 1.0→1.05，周期 3s，ease-in-out | AI Avatar 状态指示 |
| **粒子浮现** | 随机位置，淡入 0.8s，漂移 5px | 数据点装饰 |
| **卡片悬浮** | translateY -4px，shadow 扩散，0.3s ease | 卡片 hover |
| **渐变流动** | 色相旋转 360°，周期 20s，线性 | 强调按钮背景 |
| **打字机** | 字符逐个显示，间隔 30ms | 对话演示区 |

## 6. 各模块详细设计

### 6.1 Hero Section（首屏）

**布局**：全屏高度，居中布局，背景为动态数据粒子场

**背景效果**：
- 深空黑底色（#0A0E1A）
- 粒子系统：200-300个微小光点（#00D4FF，透明度 0.2-0.6），缓慢漂移
- 3-5条细光线（1px，渐变 #00D4FF → transparent）从底部向上流动，模拟数据流
- 中央偏上：一个发光的核心球体（径向渐变，#00D4FF 中心 → transparent），缓慢呼吸缩放

**内容结构**：

```

[中央内容区]

   通过对话，让数据库运营、知识获取变得像聊天一样简单
```

- **次按钮**：圆角 12px，边框 1px solid #00D4FF，背景透明，文字 #00D4FF

**底部提示**：

## 8. 技术实现建议

### 8.1 前端技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **框架** | Next.js 14 / React 18 | SSR 优化首屏，App Router |
| **样式** | Tailwind CSS + CSS Modules | 原子化 + 组件级隔离 |
| **动效** | Framer Motion + GSAP | 滚动触发、复杂序列动画 |
| **粒子** | Canvas 2D / Three.js (轻量) | 背景粒子场，300粒子以内用 Canvas |
| **图标** | Lucide React | 统一图标库 |
| **字体** | Inter (Google Fonts) + JetBrains Mono | 本地加载或 next/font |

### 8.2 性能指标

| 指标 | 目标 | 策略 |
|------|------|------|
| **LCP** | < 2.5s | 首屏内容优先加载，字体预加载 |
| **FID** | < 100ms | 事件处理优化，避免长任务 |
| **CLS** | < 0.1 | 图片/卡片预设尺寸，避免布局跳动 |
| **动画帧率** | 60fps | 使用 transform/opacity，避免 layout thrashing |
| **首屏体积** | < 200KB | 代码分割，按需加载非首屏内容 |

### 8.3 动画实现要点

**背景粒子系统**：
```javascript
// 伪代码
const particles = Array(300).fill().map(() => ({
  x: random(width), y: random(height),
  vx: random(-0.5, 0.5), vy: random(-0.5, 0.5),
  size: random(1, 3), opacity: random(0.2, 0.6)
}));

// 每帧更新
function update() {
  particles.forEach(p => {
    p.x += p.vx; p.y += p.vy;
    if (p.x < 0) p.x = width; // 边界循环
    // 绘制
    ctx.fillStyle = `rgba(0,212,255,${p.opacity})`;
    ctx.fillRect(p.x, p.y, p.size, p.size);
  });
  requestAnimationFrame(update);
}
```
